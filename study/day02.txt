day 02

1. <!DOCTYPE html> : 현재 문서가 HTML5 언어로 작성한 웹 문서라는 뜻
2. <html>~</html> : 웹 문서의 시작과 끝을 나타내는 태그
- <html lang = "ko"> 를 사용하는 이유 -> 
- 1. 검색 시 '한국어로 된 문서'로 범위를 제한할 경우, <html lang = "ko">인 문서를 우선 검색함
- 2. 화면 낭독기에서 웹 문서를 소리 내어 읽어 줄 때 해당 언어에 맞추어 발음이나 억양, 목소리 등을 다르게 할 수 있음
3. <head>~</head> : 웹 브라우저가 웹 문서를 해석하는 데 필요한 정보를 입력
- <meta> 태그 : 웹 문서와 관련된 정보를 지정할 때 사용(웹 브라우저에는 보이지x)
- <meta charset = "UTF-8"> : 웹 서버는 영어가 기본이므로 화면에 한글로 된 내용을 표시할 시 반드시 작성해야 함
- <title> 태그 => 웹 브라우저의 제목 표시줄에 표시됨
4. <body>~</body> : 실제로 웹 브라우저 화면에 나타나는 내용 

시맨틱 태그 : 이름만으로도 어떤 역할을 하는지 알 수 있음
ex) 텍스트 단락을 줄인 <p> 태그나 앵커를 줄인 <a> 태그 등
- 왜 필요?
1. 웹 브라우저가 HTML의 소스 코드만 보고도 어느 부분에 제목인지 메뉴인지 본문 내용인지 쉽게 알 수 있음
2. 문서 구조가 정확히 나눠지므로 여러 스마트 기기의 다양한 화면에서 웹 문서를 표현하기가 쉬움
3. 웹 사이트를 검색할 때 필요한 내용을 정확히 찾을 수 있음
- 주요 시맨틱 태그
1. <header> 태그 = 헤더 영역]
    - 사이트 전체 또는 특정 영역의 헤더가 있음
	- 사이트에서 주로 맨 위쪽이나 왼쪽에 있으며, 검색 창이나 사이트 메뉴를 삽입
2. <nav> 태그 = 네비게이션 영역
	- 같은 웹 문서 안에서 다른 위치로 연결 또는 다른 웹 분서로 연결하는 링크를 만듦
	- 웹 문서의 위치에 영향을 받지 않으므로
	- 헤더나 푸터, 사이드 바 안에 포함될 수도 있고 독립해서 사용할 수 있음
3. <main> 태그 = 핵심 콘텐츠
	- 핵심이 되는 내용을 넣음(페이지마다 똑같이 들어간 정보는 넣을 수 X)
	- 웹 분서에서 한 번만 사용가능
4. <article> 태그 = 독립적인 콘텐츠
	- '블로그의 포스트'나 '뉴스 사이트의 기사'처럼 독립된 콘텐츠 항목
 	- 태그 여러 개 사용가능하고, 이 안에 <section> 태그도 넣을 수 있음
5. <section> 태그 = 콘텐츠 영역
	- 몇 개의 콘텐츠를 묶는 용도로 사용
6. <aside> 태그 = 사이드 바 영역
	- 본문 내용 외에 왼쪽, 오른쪽, 혹은 아래쪽에 사이드 바를 만듦
7. <footer> 태그 = 푸터 영역
	- 웹 분서 맨 아래쪽에 있는 푸터 영역을 만듦
	- 사이트 제작 정보, 저작권 정보, 연락처, SNS링크 등의 정보를 넣음
8. <div> 태그 = 여러 소스를 묶음
	- <div id="header">나 <div class="detail">처럼 id나 class 속성을 사용해서 문서 구조를 만들거나 스타일을 적용할 때 사용
